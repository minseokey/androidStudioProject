20191637이민석 구현기능 설명

1. 첫번째 화면 (Relative Layout 사용)
 1-1) 화면 가운데에 배치한 3페이지 상품정보 페이지로 이동한다.(Button)
 1-2) 로그인 버튼은 중앙 하단에 위치, 전역변수내에 존재하지 않는 값을 입력하면 토스트 메시지로
"다시 시도해주세요" 라고 출력함. 만약 전역변수 내에 존재하는 회원정보를 입력한다면 3페이지로 이동한다(intent,activity이용).
여기서 비밀번호 입력창(EditText의 inputType는 "textPassword"으로 하여 값을 입력하면 '*'로 보이도록 만들었다.
 1-3) 회원가입(register)버튼을 누르면 회원가입페이지인 2페이지로 이동한다(intent,activity이용). 
 1-4) 알맞는 ID,Password를 입력하면 회원정보의 Parcelable 인터페이스를 활용하여 세번째 페이지에서 ParcelableExtra로 회원의 정보를 받아올 수 있도록 하였습니다(유저관리에서 설명). 

	
2. 두번째 화면 (Linear Layout 사용)
 2-1) 회원가입창은 ID, Password, check Password, Name, Phone, Address 의 총 6개로 이루어져있다. 
각각의 inputType은 Password, check Password는textPassword로, Phone은 phone로 받았다. 각각의 EditText칸에는 hint속성으로 어떤 값을 넣어야 할지가 나와있다.
 2-2) 각각의 input의 data validation은 정규식을 이용하여 하였고 java파일의 Validations에 구현이 되어있다.
비밀번호는 8~20 자리의 숫자, 알파펫, 특수기호를 받으며, ID는 6~15자리의 숫자, 알파벳, 특수기호를 받는다.
phone은 -를 제외한 11자리를 받아서 사용한다.
2-3) 간략약관은 스크롤을 가능하게하는 scrollbars 요소를 추가하여 
약관 동의, 비동의 버튼은 라디오 그룹의 라디오 버튼 위젯을 사용하여 구현하였습니다.
2-4) 회원정보에 저장된 여려 계정 정보에 있는 ID값과 현재 내가 넣은 아이디 값을 비교하여 중복되는 아이디인지, 정규식에 들어있는 정보와 일치하는지
여부를 조사하여 ID확인 버튼을 누르면 "ID가 확인되지 않습니다.ID확인을 해주세요" 이라는 멘트를 띄웁니다.
 

3. 세번째 화면 (Grid Layout사용) 
 3-1) 상품명, 상품이미지를 띄우는 페이지를 Linear Layout 내부에 스크롤이 가능하도록 스크롤 뷰를 배치하고 그 안에 Grid Layout 을 넣어서 만들었습니다. 
이 그리드 레이아웃 내부에 각각 orientation을 horizontal로 만들어서 상품이미지(imageView) 와 상품명(TextView) 을 가로로 배치했습니다.여러가지 상품을 띄워도 스크롤로 내려가면서 상품을 다 볼 수 있도록 하였습니다.
 3-2) 세번째 화면 하단에 버튼을 추가하여 회원정보를 볼 수 있게 하였습니다. 로그인이 되어있는 상태에서 회원정보를 누르면 getParcelableExtra로 받아온 객체의 이름과 ID를 Toast메세지로 띄워주고 만약 회원가입이 안되어 있다면
회원가입을 권유하는 네번째 페이지로 이동하도록 하였습니다.
 3-3) 상품추가 버튼은 만들었으나 기능은 구현하지 못하였습니다...


4. 네번째 화면(회원가입 권유 페이지, Linear Layout, visibility 요소 사용)
 4-1) 우선 회원가입 페이지로 갈건지 말건지 버튼이 있는 레이아웃은(Linear Layout) visibility 요소를 invisible로 주어서 안보이게 만들어 놓습니다.
 4-2) 사용자가 회원가입이 되어있는 상태에서 로그인을 안하고 상품페이지로 간것인지 아님 회원가입이 안되어있는 상태인지 확인합니다.
만약 회원가입이 되어 있다면 첫번째 페이지로 이동시킵니다(단방향 intent 이용). 만약 회원가입이 안되어있는 상태라면 아까전에 가려둔 두번째 레이아웃을 띄워줍니다.
 4-3) 이 레이아웃에는 회원가입을 할 것인지, 아님 그냥 돌아가서 상품을 볼 것인지 물어보는 버튼을 두개 띄워줍니다.
돌아가기 버튼을 누르면 (단방향 intent) 를 통해 세번째 페이지로 돌아가고 회원가입 버튼을 누르면 (단방향 intent)를 통해 두번째 페이지로간다.

5. 유저관리 (전역변수 사용)
 5-1) 유저는 전역변수로 구현했고 User 클래스를 통해 각각의 유저를 만들어주고 그걸 UserSet 클래스의 static ArrayList인 userSet 에 저장하여 관리한다 
 5-2) User 객체에 저장되는 정보로는 userID, userPassword, userName, userPhone, userAddress가 있다. 
 5-3) 각 액티비티에 객체를 전달해주기위한 방법으로는 Parcelable interface 를 사용했다. 생성자인 User(Parcel in) 메소드를 사용하기 위해서 Creator라는 필드를 선언해주고 인텐트로 주고받을때
로그인시에 putExtra에 객체를 넣어 보내주고 받을때 ParcelableExtra를 이영하여 받는다. 이렇게 각 액티비티간에 intent 로 객체를 넘겨 받아 로그인된 유저 정보를 첫번째 로그인 페이지에서 세번째 상품정보 페이지까지 가져갈 수 있었다.
이 정보를 이용하여 세번째 페이지의 로그인 여부를 판별하여 정보를 띄워주거나, 회원가입 권유를 할 수 있었다.
